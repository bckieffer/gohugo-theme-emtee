# emtee

A minimal Hugo theme for personal blogs. Single-author, semantic HTML, WCAG 2.1 AA accessible, no JavaScript, no build dependencies beyond Hugo itself.

---

## Requirements

Hugo **≥ 0.146.0** — required for `layouts/_partials/` resolution.

---

## Installation

```bash
git submodule add https://github.com/bckieffer/gohugo-theme-emtee themes/emtee
```

Set `theme = "emtee"` in `hugo.toml`.

---

## Configuration

All params are optional; defaults are shown.

```toml
[params]
description   = ""          # Site-wide meta description (fallback for pages without one)
mainSections  = ["posts"]   # Sections counted as blog posts
author        = ""          # Author name — appears in footer and OG meta
authorBio     = ""          # Short bio — appears in footer
ogImage       = ""          # Root-relative path to fallback social-card image (1200×630)
twitterHandle = ""          # e.g. "@example" — omit or leave empty to suppress twitter:site
```

Recommended `hugo.toml` additions that the theme expects to be set at the site level:

```toml
locale = "en-us"
enableRobotsTXT = true

[frontmatter]
date        = ["date", ":fileModTime", ":default"]
publishDate = ["publishDate", "date", ":fileModTime", ":default"]
lastmod     = ["lastmod", "date"]

[permalinks]
posts = "/:year/:month/:day/:slug/"

[pagination]
pagerSize = 20
path      = "page"

[markup.goldmark.renderer]
unsafe = false

[services.rss]
limit = 20
```

## Content Structure

Posts live in `content/posts/` as a flat directory of slug-named Markdown files. The publish date is in front matter, not the filename.

```
content/
+-- posts/
|   +-- _index.md          # Suppress /posts/ listing (_build: render: never)
|   +-- hello-world.md
|   +-- my-second-post.md
+-- 2024/                  # Archive stubs — generated by scripts/gen-archives.sh
    +-- _index.md
    +-- 01/
        +-- _index.md
```

### Post Front Matter

```toml
+++
title = 'Hello World'
date = 2024-01-15T09:30:00-05:00
description = ''         # Used in meta description and post summaries
tags = []
draft = true
# slug = ''              # Defaults to filename without extension
# image = ''             # Root-relative path; overrides ogImage for this post
+++
```

Create a new post with the built-in archetype:

```bash
hugo new posts/my-post-slug.md
```

Slug convention: lowercase ASCII letters, digits, and hyphens only (`^[a-z0-9]+(-[a-z0-9]+)*\.md$`).

## Archive Pages

The theme provides `scripts/gen-archives.sh` to generate the `_index.md` stubs that back `/YYYY/` and `/YYYY/MM/` archive URLs. Run it from the site root before every build:

```bash
./themes/emtee/scripts/gen-archives.sh
```

The theme also provides `scripts/lint-slugs.sh` to enforce the filename convention before a build.

## Templates

| URL pattern | Template |
|---|---|
| `/` | `layouts/home.html` |
| `/YYYY/`, `/YYYY/MM/` | `layouts/section.html` |
| `/YYYY/MM/DD/slug/` | `layouts/page.html` |
| `/tags/` | `layouts/taxonomy.html` |
| `/tags/<tag>/` | `layouts/term.html` |
| `/robots.txt` | `layouts/robots.txt` |
| unmatched | `layouts/404.html` |

Override any template by placing a file at the same path under your site root.

## Layout

Two-column desktop (two `400px` columns, `12px` gutter, `812px` total), single-column mobile (below `768px`). No wrapper `<div>` — the CSS Grid targets `<body>` directly; `<header>`, `<main>`, `<aside>`, and `<footer>` are the grid items.

---

## Accessibility

- WCAG 2.1 Level AA target
- Skip-to-content link as first focusable element on every page
- `:focus-visible` keyboard indicator; pointer clicks do not trigger outlines
- `prefers-reduced-motion` respected for all transitions
- `prefers-color-scheme` dark mode with independently verified contrast ratios
- All ARIA landmark roles provided by native semantic elements — no `role` overrides

---

## Styling

Single stylesheet at `assets/css/main.css`, processed through Hugo Pipes (`Minify` + `Fingerprint`). Plain CSS with custom properties — no preprocessor, no build step.

Color tokens:

```css
:root {
  --bg:       #ffffff;
  --fg:       #1a1a1a;
  --fg-muted: #555555;
  --link:     #0050ff;
  --border:   #e5e5e5;
}
```

Override any token by adding a `:root` block to your site's own stylesheet and importing it after the theme's.
